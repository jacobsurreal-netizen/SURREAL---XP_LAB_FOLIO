# AI OPS DOCUMENT HEADER STANDARD
Version: v0.1
Status: Working Standard
Last Updated: 2026-03-09

---

## Purpose

Provide a consistent metadata header for all AI OPS documents
so AI assistants can immediately understand the document's authority,
scope, and dependencies.

---

## Standard Header Template

```
DOC_ID:
TITLE:
VERSION:
STATUS:
SYSTEM:
LAYER:
SCOPE:
DEPENDS_ON:
SUPERSEDES:
LAST_UPDATED:
AUTHOR:
```

---

## Status Definitions

Authoritative – Current system law  
Operational – Active development guidance  
Planning – Recommended sequencing or roadmap  
Draft – Forward‑looking design exploration  
Template – Reusable documentation scaffold  

---

## Layer Definitions

Knowledge – conceptual documentation and philosophy  
Pipeline – asset preparation and orchestration tools  
Runtime – browser runtime systems  
Experience – visual and interaction design rules

---

## Recommended File Prefixes

ADR_ – Architecture Decision Record  
ARCH_ – architecture maps and system relationships  
SPEC_ – subsystem technical specifications  
GUIDE_ – workflow or operational guidance  
STATUS_ – live system state snapshots  
TEMPLATE_ – reusable document templates

---

### Language Note

Documentation is written in English for AI compatibility.
Active development conversations occur primarily in Czech.
