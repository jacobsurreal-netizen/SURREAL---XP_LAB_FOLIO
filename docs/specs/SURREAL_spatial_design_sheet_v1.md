# 1. Core worldview

## Projekt není jen portfolio
Projekt plní několik úloh najednou:
- skill proof
- experimentální laboratoř
- portfolio
- showroom spatial webu
- reference implementation pro budoucí template extraction

## Hlavní mental model
Visitor není jen user.
Visitor je spíš **operátor sondy**.

### Model světa
- **Artefakt** = entita / signál / zhmotněná idea
- **HUD** = jazyk a překladač systému
- **Visitor** = sonda / operátor
- **Scroll** = pohyb prostorem
- **CTA** = navázání komunikace

# 2. Artifact principles

## Artefakt není tlačítko
Artefakt má působit jako:
- entita
- anchor
- nositel významu
- tiché centrum světa

Nemá se chovat jako hover button.

## Dlouhodobý směr artefaktu
- trojúhelníkový charakter
- směřování ke glyph / Penrose-like logice
- vibe: sci-fi ambient, ne jen dark void cyberpunk
- reference vibe: Arrival / Invasion / ambient spatial installation

## Evolution model
- v1: placeholder relic / unknown signal object
- v2: více strukturovaná signal forma
- v3: glyph emergence + transformace světa

# 3. Spatial sound ideas

## Sound role
Zvuk není hudba.
Zvuk je behaviorální vrstva světa.

## Dramaturgický zvukový oblouk
### Hero / Void
- deep space drone
- velmi tichý
- pocit unknown signal

### Exploration
- drone + lehké harmonické náznaky
- artifact / node focus může lehce proměnit charakter

### CTA
- drone se čistí
- přechází do harmonického ambientního pole
- psychologický efekt: contact established / signal resolved

## Sound behavior ideas
- volume podle vzdálenosti kamery od artefaktu
- lowpass / clarity podle distance
- jemný pitch drift podle scroll progress
- CTA crossfade drone → harmonic ambience

## UX pravidlo
Sound musí být:
- opt-in
- jemný
- skoro neviditelný
- návštěvník má cítit atmosféru, ne „super soundtrack“

# 4. Project objects / portfolio roadmap

## V1
Projektová sekce = prostorový kruhový carousel kolem artefaktu.

### Princip
- max 5 projektů
- orbiting project objects / cards
- filmový parallax / holografický look
- focus state
- metadata v HUD
- click-through na projekt

## V2
Project node → mini showcase / vitrína za sklem.

### Princip
- jednoduchá showcase scéna
- minimální interakce
- 1–2 signální efekty z projektu
- link na live projekt / case page

## V3
Project node → portál do mini světa / spatial case study.

## Hard limit počtu projektů
- v1: do 5
- v2 / v3: ideálně do 10
- absolutní strop: 12

Důvod:
Spatial svět není grid karet. Každý projekt je objekt ve světě.

# 7. No-Go zones

## NGZ 1 — Efektová spirála
Příliš mnoho shaderů, particles, bloom, overlays.

## NGZ 2 — Interakční chaos
Příliš mnoho input modalit.

## NGZ 3 — Moc objektů
Void ztrácí sílu, artefakt přestane být centrum.

## NGZ 4 — HUD jako ornament
HUD má být jazyk, ne sci-fi samolepka.

## NGZ 5 — Lore bez funkce
Pojmy musí reprezentovat reálnou funkci v systému.

## NGZ 6 — Performance peklo
Spatial svět musí zůstat plynulý.

## NGZ 7 — Architektonický chaos
Subsystems se nesmí začít nekontrolovaně proplétat.
