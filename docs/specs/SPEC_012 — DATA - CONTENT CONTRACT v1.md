# SPEC_012 — DATA / CONTENT CONTRACT v1

Status: PROPOSAL

---

# PURPOSE

Define minimal structure for system data.

---

# CORE RULE

Data must be simple, consistent, portable.

---

# PROJECT DATA

Each project must include:

* id
* title
* description
* visual reference
* optional metadata

---

# CTA DATA

* message
* state (idle / active / sent)

---

# SOUND DATA

* profile id
* enabled flag

---

# RULES

* no complex nested structures
* no UI-driven data
* data must be reusable across systems

---

# DESIGN GOAL

Data should support system, not dictate it.

---

END
