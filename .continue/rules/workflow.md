# Working style

Before editing:
1. Summarize the bug or task in one paragraph.
2. List the files you need to inspect.
3. Propose the smallest safe fix.

When editing:
- Avoid unrelated formatting changes.
- Keep imports sorted only if the edited file already follows that style.
- Explain why each changed file was necessary.

After editing:
- Suggest a commit message in conventional commits format.
- Suggest a quick manual test plan.
